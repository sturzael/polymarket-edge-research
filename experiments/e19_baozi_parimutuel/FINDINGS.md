## 2. V4 vs V5 discrepancy (confirmed, user-flagged arb)

- **V4** (current, program `FWy…ruJ` V4.7.6) = pari-mutuel pool mechanics, no AMM.
- **V5** (development, per Baozi's own content) = Polymarket-style conditional YES/NO tokens traded on **Orca Whirlpools** (Solana's concentrated-liquidity AMM).
- Running side-by-side creates real arb: V4 pari-mutuel close price (`yes_pool / total`) can diverge from V5 Orca AMM mid. V4 only updates on each bet's ratio effect; V5 is continuous CL-AMM. Convergence requires an arb agent.
- Not yet directly observable by this agent because V5 Orca pool addresses weren't exposed to WebFetch. Capturing the spread requires: (1) V4 state via `probe_baozi.py`, (2) V5 Whirlpool address for the same market's YES token, (3) Orca SDK query for current tick/price, (4) diff.

## 3. Data-access methodology (runnable)

No REST API → use JSON-RPC `getProgramAccounts` with memcmp on the MARKET discriminator. `scripts/probe_baozi.py` does this end-to-end and writes `data/baozi_markets.parquet`. Public `api.mainnet-beta.solana.com` usually allows this but is rate-limited; if 429/403, point `--rpc` at Helius or Triton.

## 4. T-7d snapshot — why we don't have one (yet)

`getProgramAccounts` returns **current** account state. For resolved markets that's T-0 (final pool), not T-7d. To get T-7d:
1. `getSignaturesForAddress` on each market PDA → every tx that touched it
2. Walk backward until `block_time ≤ closingTime - 7×86400`
3. Decode each bet tx and replay to reconstruct `yes_pool(t)`, `no_pool(t)`
4. Snapshot `implied_yes(t=close-7d)`

Solana public RPC keeps tx history ~2 weeks for unknown accounts; older markets need Helius/QuickNode archive or a Substream indexer. Not done here.

**Consequence:** calibration is **at-close**, not T-7d. Polymarket baseline is T-7d. Close has more info ⇒ typically better calibration ⇒ less FLB. So: **if Baozi at-close shows MORE FLB than Polymarket T-7d, that's a strong pari-mutuel signal. If similar or less, inconclusive.**

## 8. A-priori expectation

Pari-mutuel markets are textbook-known for stronger FLB than book-based markets (Thaler & Ziemba 1988 et seq.). Reasons: no MM arb to flatten the curve, late money can't revise earlier implied odds (only reinforces crowd bias), small pools mean single bets move implied prob 5-10pp.

**Prediction:** if Baozi has meaningful sports volume, sports 0.55-0.60 would probably show **+35-50pp** deviation with wide CIs due to small n. At-close-vs-T-7d time-mismatch works the other way; net ambiguous without the number.
