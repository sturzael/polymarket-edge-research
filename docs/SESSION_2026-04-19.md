## What's next, in priority order

### P1 — Forward-validation collector for calibration (~2 hrs code, 30 days wait)

Build a launchd agent that logs (every hour) active sports markets with 6.5-7.5 days to resolution. Capture `price`, `bestBid`, `bestAsk`, `category`, `condition_id`, `timestamp`. Wait for resolutions. Bucket and compare to the [e16 FINDINGS historical table](../experiments/e16_calibration_study/FINDINGS.md#2e-deep-sports-only-pull-with-liquidity-n2025). If forward yes_rate per bucket matches historical within ~5pp, bias persists → strategy is real and can be deployed.

This is the one unlocking step. Nothing else moves forward until it runs.

### P2 — Observe existing paper-trade through resolution (28-63 days, passive)

First resolution 2026-05-18 (NBA Rookie of the Year, $16 position). Full basket 2026-06-22. Will tell us:
- Whether neg-risk arb class survives at $5k-$20k paper sizing
- What fraction of "qualifying" entries were phantoms (via phantom_audit + resolution mismatch)
- Annualized returns per hold-day (after fees, after phantom loss)

### P3 — Follow-up code enhancements (optional, when paper-trade signal clarifies)

- Per-position edge-trajectory tracking in paper_trader (so we can see decay BEFORE resolution, same as forward_trader does for the 6 picked events).
- Raise MAX_PER_POSITION_USD beyond $500 for the calibration strategy (different economics — 20pp edges can take more size).
- Scanner-driven calibration-entry mode (analogous to paper_trader but for the sports-favorites strategy, once P1 validates it).

### P4 — VPS decision

Defer until P1 or P2 produces signal worth acting on. VPS is ~$6/mo and adds ops burden. Laptop under caffeinate is fine for paper-trade observation.
